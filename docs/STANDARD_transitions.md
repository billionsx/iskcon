## ЧАСТЬ 1. КАК УСТРОЕНО СЕЙЧАС

### 1.1. Три слоя

```
┌─────────────────────────────────────────┐
│  ОВЕРЛЕЙ  (36 штук)                     │  ← перекрывает всё
│  книга · личность · джапа · корзина …   │
├─────────────────────────────────────────┤
│  ВИТРИНА / РАЗДЕЛ  (Tier-1)             │  ← вкладка внутри зала
│  Личности · Книги · Бхаджаны …          │
├─────────────────────────────────────────┤
│  ОСНОВА  (Tier-0) — ПЯТЬ вкладок        │  ← нижнее меню
│  Садхана · Кришна · Гауранга ·          │
│  ИСККОН · Богатства                     │
└─────────────────────────────────────────┘
```

### 1.2. Основа — пять вкладок, других нет

| Вкладка | Что внутри |
|---|---|
| **Садхана** | Лента · Практика · Календарь · Кабинет |
| **Кришна** | досье: Таттва · Нама · Рупа · Гуна · Парикары · Лила · Дхама |
| **Гауранга** | досье: те же уровни |
| **ИСККОН** | ИСККОН · Новости · Центры · Рестораны · Образование · Структура · Документы · Ссылки |
| **Богатства** | Личности · Книги · Бхаджаны · Киртаны · Прасад · Дхама |

`setTab` с любым другим значением = **белый экран** (ЗКН-Н027).

### 1.3. Один владелец истории — и это работает

`App` — **единственный**, кто слушает `popstate` (ЗКН-Н002). Адрес пишется
только через `nav.ts`. Компоненты **подписываются** (`subscribeNav`), а не
заводят свой слушатель.

### 1.4. Как адрес превращается в экран

```
клик → navigate(path)
         ├─ pushUrl(path)          запись в историю
         └─ applyPath(path)
              ├─ СБРОС всех 36 оверлеев     ← верно: экран не наследует мусор
              ├─ setTab(...)                ← какая ОСНОВА
              └─ setOpen…(…)                ← какой ОВЕРЛЕЙ
```

## ЧАСТЬ 2. ЧТО СЛОМАНО

### 2.1. ⛔ КОРЕНЬ: 25 из 42 адресов НЕ ОБЪЯВЛЯЮТ ОСНОВУ

Главная беда. Адрес открывает оверлей, но **не говорит, что под ним**.

| Адрес | Открывает | Основа под ним |
|---|---|---|
| `/japa` | Счётчик джапы | **что было** |
| `/story` | Дневник садханы | **что было** |
| `/verse` | Стих дня | **что было** |
| `/promise` | Обеты | **что было** |
| `/darshan` | Даршан | **что было** |
| `/progress` | Мой прогресс | **что было** |
| `/ekadashi` | Экадаши | **что было** |
| `/favorites` · `/search` | Избранное · Поиск | **что было** |
| `/notes` · `/note/<id>` · `/cart` | Заметки · Корзина | **что было** |
| `/iskcon/centers` · `/my/centers` | Центры | **что было** |
| `/centers/review` | Модерация | **что было** |
| `/prasad/book` | Книга прасада | **что было** |
| `/post/<id>` · `/dasa/<id>` | Пост · контент | **что было** |
| `/admin` · `/downloader` · `/read` | Служебное | **что было** |

**Что это значит на деле.**

Человек стоит на «Богатства → Книги». Жмёт «Джапа» — открывается счётчик.
Закрывает — попадает **в Книги**, хотя джапа это Садхана.

Свежий заход по ссылке `/japa` (закладка, QR): основы нет — под оверлеем
**пустота или случайный экран**.

**Оверлей без основы — это оверлей над неизвестностью.**

### 2.2. ⛔ «Назад» неотличим от «Закрыть»

`goBack()` = `history.back()` — **шаг по истории**, а не «снять оверлей».

Видно, когда пришёл **по прямой ссылке**: истории нет, `back()` уводит **из
приложения**. Крестик должен возвращать к основе, а не выкидывать в браузер.

### 2.3. ⚠️ Оверлеи не знают друг о друге

36 состояний `openX`. Внутри одного адреса могут открыться два сразу. Кто выше —
решает порядок в JSX, а не смысл.

### 2.4. ⚠️ Подтаб хранится в компоненте, а не в адресе

Смена царства уносила подтаб с собой (лечено `key`, ЗКН-Н028). Но корень глубже:
подтаб — **состояние компонента**, а должен выводиться из адреса.

### 2.5. ⚠️ Корни вне стандарта

`/read` · `/dasa` · `/post` · `/place` · `/doc` — пять корней, о которых
ничего не сказано.

## ЧАСТЬ 3. КАК ДОЛЖНО БЫТЬ

### ЗАКОН 1. КАЖДЫЙ АДРЕС ОБЪЯВЛЯЕТ ОСНОВУ

Адрес — **полное описание экрана**, а не подсказка. Отвечает на два вопроса:

```
   ОСНОВА:  какая из пяти вкладок под низом
   ОВЕРЛЕЙ: что сверху (или ничего)
```

**Различение, которое всё решает:**

- **Оверлей РАЗДЕЛА** (джапа, центры, читалка, рецепт) — приходит **со своей
  основой**. Закрыл — попал в свой раздел.
- **Оверлей ПРИЛОЖЕНИЯ** (поиск, избранное, корзина, заметки) — надстройка над
  ЛЮБЫМ экраном, **основу не меняет**. Закрыл — вернулся туда, где стоял.

Сейчас они смешаны — оттого поведение непредсказуемо.

| Адрес | Основа | Оверлей |
|---|---|---|
| `/japa` `/story` `/verse` `/promise` `/darshan` `/progress` `/ekadashi` | **Садхана** | инструмент |
| `/calendar` `/id` `/sadhana` | **Садхана** | — |
| `/hero` `/books` `/bhajans` `/kirtans` `/prasad` `/dhama` | **Богатства** | — |
| `/<книга>` `/<книга>/<гл>/<стих>` | **Богатства** | Читалка |
| `/prasad/<рецепт>` `/prasad/book` | **Богатства** | Рецепт · Книга |
| `/iskcon` `/iskcon/*` `/my/centers` | **ИСККОН** | Центры и пр. |
| `/krishna/*` | **Кришна** | — |
| `/gauranga/*` | **Гауранга** | — |
| `/<личность>` | **не меняет** | Карточка |
| `/search` `/favorites` `/notes` `/cart` `/donate` | **не меняет** | Оверлей приложения |

### ЗАКОН 2. «НАЗАД» И «ЗАКРЫТЬ» — РАЗНОЕ

- **«Назад»** (жест, кнопка браузера) → шаг по истории.
- **«Закрыть»** (крестик) → **снять оверлей, показать основу**.

Истории нет (пришёл по ссылке) → «закрыть» ведёт **к основе этого адреса**,
а не из приложения.

### ЗАКОН 3. ПОДТАБ ВЫВОДИТСЯ ИЗ АДРЕСА

`/gauranga/guna/milost` — подтаб **в адресе**. Компонент его **читает**, а не
помнит. Смена царства не может унести подтаб: адрес другой — подтаб другой.

### ЗАКОН 4. ОДИН ОВЕРЛЕЙ РАЗДЕЛА ЗА РАЗ

Открывается новый — предыдущий закрывается. Стопка оверлеев = лабиринт.
Оверлей ПРИЛОЖЕНИЯ может лечь поверх — но и он один.

## ЧАСТЬ 4. ЧТО ВНЕДРЕНО

1. **Таблица `SCREENS`** — единственный источник: адрес → (основа, оверлей).
2. **`setTab` ОБЯЗАТЕЛЕН** в каждой ветке `applyPath`. Гейт валит сборку,
   если ветка не объявила основу (ЗКН-Н029).
3. **`closeOverlay()`** отдельно от `goBack()` (ЗКН-Н030).
