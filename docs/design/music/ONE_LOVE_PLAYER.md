# ONE LOVE PLAYER — архитектура /music под Книги · Лекции · Киртаны · Бхаджаны
Стандарт: Apple Music iOS 26 (LAW_MUSIC.md, ЗКН-Д029). Дата: 22.07.2026.

## 0. Принцип
/music перестаёт быть демо-копией и становится ЕДИНЫМ аудио-хабом приложения.
Вся геометрия, материалы, типороли, анимации — из LAW_MUSIC без изменений.
Меняются только СЛОВАРЬ (разделы) и ДАННЫЕ (наши источники вместо демо).

## 1. Инвентарь функций боевого плеера (src/player), которых нет в /music
| Функция | Где живёт | Судьба в /music |
|---|---|---|
| Скорость 1/1.25/1.5/2 | store.PLAYER_RATES, cycleRate | ⋯-меню плеера + бейдж «1.5×» у времени (паттерн AM Podcasts) |
| Таймер сна (мин / «после записи») | setSleep, sleepEnd | ⋯-меню → шторка-лист (наша sheet R40) |
| Резюме позиции (persist 5 c) | PERSIST_KEY iol.player.v2 | глобально; карточки «Продолжить» на корнях |
| Режим книги plain/commentary | AudioMode, setMode | тумблер-капсула на странице книги + в плеере (pl-tog) |
| Сквозной переход канто | advanceScope | остаётся в сторе, UI не нужен |
| «К тексту / Читать» (ЗКН-Б011) | NowPlaying openText | кнопка нижнего ряда плеера (слот лирики) |
| Скачать mp3 | download() | ⋯-меню записи и плеера |
| Избранное (единый ключ fav) | mediaTrackKey | звезда в шапке плеера (уже есть) + «Моё» |
| Очередь: порядок fwd/shuffle/rev, повтор, группы | buildOrder, cycleRepeat | наш вид «Далее» (pl-tog) — движок общий |
| MediaSession | updateMediaSession | остаётся |
| Катха: 5 спикеров, коллекции, сортировки, q:/fav: deep-link | AudioLibrary | раздел «Лекции» |

## 2. Навигация
Док (геометрия без изменений: вкладка 87.9×44.6, знак 26, подпись 10, активная —
красный глиф без подложки; поиск-круг d62 справа):
**Книги · Лекции · Киртаны · Бхаджаны + Поиск**.

### 3.5 Поиск
Наше поле в доке + фокус-состояние (2375: x20…373, h36). Результаты секциями:
Top result (big 88/102) → Спикеры (круглые) → Книги/главы → Киртаны → Бхаджаны;
недавние запросы; deep-link q:/fav: сохраняются.

## 4. Данные и мост
- Единый движок: PlayerProvider (src/player/store) становится провайдером и для
  /music; amx-плеер читает те же track/queue/rate/sleep. Демо-данные ALL_SONGS
  удаляются. Source: book|katha|kirtan|bhajan — уже совпадает с разделами.
- Каталог лекций: применить Ц12 (каталог без треков + SQL-поиск + постранично).
- Обложки: реальные (спикеры/книги/альбомы), заглушка — AUDIO_FALLBACK_COVER.

## 6. Проработка v2 (22.07 вечер, по правкам основателя)

### 6.1 Избранное — сердце, сквозное
- Значок: HeartIcon из ui/icons (тот же, что в шапке основного меню и в BookHeroCard).
  В шапке плеера сердце заменяет звезду (круг d32 без изменений геометрии);
  активное состояние — фирменный цвет избранного шапки.
- Запись: существующий useFavorite + mediaTrackKey/kirtanTrackKey. Категории
  FavoritesScreen уже типизированы (book/chapter/verse/katha/kirtan/bhajan) —
  трек из /music попадает и в «Избранное» раздела, и в «Моё» плеера без новой
  схемы. Ничего не дублировать: один ключ — три экрана (ЗКН-структура store:53).

### 6.2 «К тексту» — слот караоке-кнопки
- Левая кнопка нижнего ряда плеера (нынешний караоке-слот) для источника book
  становится «К тексту»: переход на ТОЧНЫЙ стих/главу/песнь через bookRef/
  scriptureRef (модуль уже строит «песнь · глава · стих»), плеер при этом
  сворачивается в мини (collapse, не dismiss) — звук продолжает идти под текстом.
- Для bhajan тот же слот открывает текст бхаджана (лирика-вид); для katha/kirtan
  без текста — кнопка в неактивном .52.

### 6.3 Мини-плеер закрывается
- Свайп-вниз по мини (>60px) = dismiss() — как в боевом MiniPlayer:59.
  Перенести жест в amx-мини; dismiss останавливает движок и убирает мини.
  Крестов не добавляем — жест-стандарт.

### 6.4 Книги-root: витрины BookHeroCard слайдерами по авторам
- BookHeroCard — готовый юнит-стандарт (ВКП/ПКП: сердце · наушники · корзина · ⋯,
  многообложечный useCoverSlider, графитовый фон). В /music Книги-root строится
  как Home→Radio Takeover: вертикальные tkbig-блоки (353, cap-скрим) —
  ПО БЛОКАМ АВТОРОВ: H2-заголовок блока = автор (с шевроном → карточка личности),
  внутри — горизонтальный слайдер витрин его книг (paged, срез 24).
- Названия книг — через renderTitle/SCRIPTURE_VOICE (курсив санскрита), не голым
  текстом.

### 6.5 Авторы = личности, сквозная синхронизация
- Каждый раздел структурируется по авторам; автор — сущность ПКЛ:
  Книги → author_entity книги; Бхаджаны → author_entity (BhajanDetail:467 ✓);
  Киртаны → artist.entityId (KirtanArtistPage:162 ✓); Лекции → спикеру добавить
  entity_id в каталог (задача данных: 5 спикеров ↔ 5 личностей).
- На страницах авторов в /music — кнопка «Страница личности →» (onOpenEntity,
  паттерн уже есть); на карточке личности — обратная полка «Аудио» его записей.

### 6.6 Новые находки исследования
- useCoverSlider: листание нескольких обложек прямо на витрине — в Книгах-root
  оставить (у многотомников живые обложки).
- scriptureRef: sub-строки глав в треклисте книги — «Песнь 1 · Глава 2 · 15–35».
- FavoritesScreen: липкий стеклянный фильтр категорий с живыми счётчиками —
  паттерн пригоден для «Моё» внутри плеера.
