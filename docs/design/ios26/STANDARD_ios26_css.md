## 0. Как читать этот документ

У каждого числа есть **происхождение**. Оно помечено:

| Метка | Что значит | Можно ли спорить |
|---|---|---|
| 📐 **ЗАМЕР** | Снято пиксельно с кадра в `refs/`. Адрес кадра указан | Нет. Это факт. Спорить можно только с методом замера |
| 🍎 **КАНОН** | Опубликованное значение Apple (HIG / системный токен) | Нет |
| ⚙️ **ВЫВОД** | Наш перевод замера/канона в CSS. Обоснование указано | Да — если предложен лучший перевод |
| 🕳 **ДОЛГ** | В референсах этого ещё нет. Число не выдумано, а **не снято** | — |

**Выдумывать число запрещено (ЗКН-БТ001).** Нет кадра — стоит 🕳, а не
правдоподобная цифра. Правдоподобная цифра хуже отсутствующей: её нельзя
проверить, и она расползается по коду как факт.

### Перепись, а не впечатление

Числа ниже сняты **счётом по всем 217 кадрам десяти продуктов**, а не взглядом
на несколько экранов. Впечатление нельзя перепроверить; перепись можно
перезапустить и получить то же:

```bash
python3 tools/assets/offload.py restore --class ios26-refs
python3 tools/assets/offload.py restore --class ios26-refs-2
python3 tools/ios26-census.py --src <папка с кадрами> --json docs/design/ios26/census.json
```

Сырой результат лежит рядом: **`census.json`** — по каждому продукту холст,
поверхности, толщина и цвет разделителя, шаг строки, врезка, плавающий бар,
акцент. Спор о числе решается перезапуском переписи, а не памятью.

### Как снимать замер, чтобы он не соврал

В таблице выше сказано: со ЗАМЕРОМ спорить можно только через метод. За один
день роспись Notes трижды упёрлась в один и тот же дефект метода, и все три
раза неверное число выглядело правдоподобно. Поэтому метод записан здесь, а не
держится в голове.

**1. Край берётся по материалу, а не по яркости.** Антиалиасинг и ореол стекла
дают около пикселя лишку, и этот пиксель решает споры. Брать надо точку, где
заливка пересекает 50 % между холстом и деталью, с долей пикселя. Что стоила
яркостная кромка: скругление «22.3 pt, не дуга» (на деле 24–26), низ панели
«21.3 / 28.3» (на деле 21.0 / 28.0), высота таб-бара «61.7 с разбросом» (на
деле 62.00 ровно).

**2. Единица замера кегля — литера, а не окно.** Прямоугольник, заданный
рукой, ловит соседний знак или срезает букву: «строка папки» вышла 48 px
вместо 43, потому что в окно попала иконка. Литера — связная область чернил,
её границы задаёт кадр. Где в строке есть знаки (меню), берётся мода высот по
всей панели: литеры кучкуются в две горки, знак в горку не попадает.

**3. У повторяющегося элемента меряется инвариант, а не следствие.** Низ
плавающей панели полгода числился «то 21.3, то 28.3» и объяснялся типом
экрана. Постоянна не кромка, а **ось**: 52.00 pt над низом экрана. Кластеров
не было — было два разных предмета, померенных за низ.

**4. Модель без соперника ничего не доказывает.** «Круг дал бы 19.3, а замер
даёт 24» — довод, у которого нет второй модели. Подгонять надо минимум двумя
и печатать ошибку обеих: дуга против суперэллипса, а не дуга против интуиции.

Метод держится инструментами, а не памятью:

| Инструмент | Что снимает | Как проверяет себя |
|---|---|---|
| `tools/ios26-corner.py` | скругление | подгоняет дугу И суперэллипс, печатает RMSE обеих |
| `tools/ios26-type.py` | cap-height | по литере или по моде панели; кегль отдаётся отдельно как ⚙️ |
| `tools/ios26-bar.py` | плавающая панель | считает ось, отсекает индикатор, клавиатуру и лист |

Адрес замера держит гейт, а не совесть: `tools/ios26-lint.py` проверяет, что
названный продукт есть в `refs/` **и что номер кадра не выходит за число кадров
продукта по реестру**. «`apple_notes` с.19» при восемнадцати кадрах роняет
прогон — раньше проходило молча, а имя-то настоящее, потому и незаметно.

🕳 **Долг:** обратное — «у каждого 📐 есть адрес» — гейтом не берётся. Адрес
законно бывает трёх видов: ссылка на кадр, продукт, названный словами
(«тумблеры», «кольцо Move в Фитнесе»), и охват («88 кадров, 7 продуктов»).
Регулярка дала 24 находки, из них большинство — нормальная проза. Шумный гейт
отключают, и тогда не работает даже та часть, что работала.

### Порядок применения
```
1. Открыть этот документ  →  число есть?  →  взять его
2. Числа нет  →  открыть refs/, снять замер  →  вписать сюда с адресом кадра
```

## 1. Система координат

| Параметр | Значение | Источник |
|---|---|---|
| Кадр референса | 1179 × 2556 px | 📐 все кадры |
| Логический холст | **393 × 852 pt** | ⚙️ 1179/3 × 2556/3 |
| Масштаб | **@3x** → `pt = px / 3` | 📐 |
| Базовая сетка | **4 pt**; половинный шаг 2 pt допустим только внутри контрола | 🍎 |
| Минимальная зона нажатия | **44 × 44 pt** | 🍎 HIG |
| Пол типографики | **11 pt** | 🍎 + ЗКН-Д006 |

В CSS `1 pt = 1 px` (CSS-пиксель), потому что браузер уже работает в логических
единицах. **Все числа стандарта пишутся в CSS как есть.** Умножать на 3 нельзя —
это была бы вёрстка в физических пикселях.

Safe area:
```css
padding-bottom: max(20px, env(safe-area-inset-bottom));
padding-top:    env(safe-area-inset-top);
```
📐 Плавающий слой держится не отступом снизу, а **осью: 52.00 pt над низом
экрана** (88 кадров, 7 продуктов). Нижняя кромка — следствие оси и высоты
детали: таб-бар высотой 62 стоит в 21.0 pt от края, панель поиска высотой 48 —
в 28.0 pt. Ось в CSS не выражается, поэтому верстаем отступ той детали, которую
делаем, и не переносим её число на другую. Подробности — 5.2.

## 2. Цвет — тёмная тема

Тёмная тема iOS 26.5 стоит на **чистом чёрном холсте**, а не на тёмно-сером.
📐 `#000000` занимает **29–59 %** площади в каждом из шести приложений.
Это не стилистика — это OLED: чёрный пиксель выключен.

### 2.1 Холсты и поверхности

| Роль | Тёмная | Светлая | Источник |
|---|---|---|---|
| Холст экрана `systemBackground` | **`#000000`** | `#FFFFFF` | 📐 все кадры |
| Поверхность карточки `secondarySystemBackground` | **`#1C1C1E`** | `#F2F2F7` | 📐 `apple_id` с.4 (чтение `#1B1B1D`, JPEG −1) |
| Приподнятая поверхность `tertiarySystemBackground` | **`#2C2C2E`** | `#FFFFFF` | 📐 `apple_calendar` с.2, 8 (10.8 % площади) |
| Материал под баром / меню | **`#181818`** непрозрачный эквивалент | — | 📐 `apple_files` с.4 таб-бар |

📐 **Перепись 217 кадров.** `#000000` — модальный холст у **всех десяти**
продуктов без исключения (доля 29–59 %). Словарь поверхностей одинаков везде:

| Поверхность | Встречается у |
|---|---|
| `#000000` холст | 10 из 10 |
| `#1C1C1E` карточка | Calendar · ID · Wallet · App Store · Notes · Fitness |
| `#2C2C2E` приподнятая | Calendar · App Store · ID |
| `#181818` стекло над чёрным | 10 из 10 (бары, меню, листы) |
| `#121212` подложка медиа | TV · Music · Dev |
| `#3A3A3A`/`#3B3B3B` заливка контрола | Music · Notes · Fitness |

**Три уровня — потолок.** Четвёртый серый в тёмной теме неразличим: между
`#2C2C2E` и `#3A3A3C` глаз на OLED разницы не видит, а иерархия ломается.
Глубже — не цветом, а **тенью и материалом** (§6).

### 2.2 Метки (текст)

🍎 КАНОН. В тёмной теме вторичные метки — **не серый цвет, а белый с альфой**:
так они одинаково читаются на любой поверхности.

| Токен | Значение | Где |
|---|---|---|
| `label` | `#FFFFFF` | Заголовок строки, крупный заголовок, значение |
| `secondaryLabel` | `rgba(235, 235, 245, 0.60)` | Подпись, метаданные, значение справа |
| `tertiaryLabel` | `rgba(235, 235, 245, 0.30)` | Шеврон, выключенное, плейсхолдер-текст |
| `quaternaryLabel` | `rgba(235, 235, 245, 0.18)` | Едва намёк; ниже — невидимо |
| `placeholderText` | `rgba(235, 235, 245, 0.30)` | Подсказка в поле ввода |

📐 Подтверждение: белый текст строки читается ровно `#FFFFFF` (`apple_id` с.4).

### 2.3 Заливки контролов

🍎 КАНОН. Серый с альфой — **не** цвет, а плёнка: работает на любом фоне.

| Токен | Значение | Где |
|---|---|---|
| `systemFill` | `rgba(120, 120, 128, 0.36)` | Крупный контрол на фоне |
| `secondarySystemFill` | `rgba(120, 120, 128, 0.32)` | Сегментированный контрол, чип |
| `tertiarySystemFill` | `rgba(118, 118, 128, 0.24)` | Поле поиска, дорожка ползунка |

### 2.4 Разделитель

| Параметр | Значение | Источник |
|---|---|---|
| Толщина | **1.0 pt** (3 физических px) | 📐 `apple_id` с.4 — ровно 3 px |
| Цвет поверх `#1C1C1E` | **`#37373A`** | 📐 три замера подряд: `#343436`, `#373739`, `#373739` |
| Канонический токен | `separator` = `rgba(84, 84, 88, 0.60)` | 🍎 — даёт `#37373A` над `#1C1C1E` ✓ |
| Непрозрачный | `opaqueSeparator` = `#38383A` | 🍎 |
| Врезка слева | **до начала текста** | 📐 68.7 pt от края экрана = 16 врезка + 16 поле + 29 плитка + 7.7 зазор |
| Врезка справа | **16 pt** от кромки карточки | 📐 линия кончается на 360.7 pt, карточка — на 377 pt |

📐 **Перепись:** толщина **1.0 pt** — модальная у восьми продуктов из десяти
(Calendar 108 · Dev 72 · Files 111 · ID 27 · Notes 74 · Wallet 22 попаданий).
Цвет над карточкой `#1C1C1E`: **`#36–38`** (`#373739` у ID, `#38383A` у Notes и
Wallet — это ровно `opaqueSeparator`). Над более светлой подложкой линия темнее
(`#292929`…`#2D2D2D`): iOS считает разделитель **альфой**, а не краской.

**0.5 px запрещён.** iOS рисует волосяную линию **в 1 pt**, а не в полпикселя;
полпикселя в браузере на 1x-экране исчезает совсем.

## 3. Типографика

### 3.1 Шрифт

```css
--font-ui:
  -apple-system, BlinkMacSystemFont,
  "SF Pro Text", "SF Pro Display", "SF Pro",
  system-ui, "Segoe UI", Roboto, "Inter", sans-serif;
```

🍎 На устройствах Apple `-apple-system` даёт настоящий SF и **сам** переключает
оптический размер: `SF Pro Text` до 20 pt, `SF Pro Display` от 20 pt.
Руками этот порог не эмулируют — это делает шрифт.

Вне Apple ближайшая метрика — **Inter**. ⚙️ Inter выше по x-height, поэтому при
подстановке кегль **не меняют**, а межстрочный держат по таблице ниже.

Цифры в счётчиках, времени, суммах:
```css
font-variant-numeric: tabular-nums;   /* иначе цифры прыгают при пересчёте */
```

| Callout | 16 | 21 | −0.31 | 400 | Текст статьи |

Трекинг в CSS — в em при данном кегле:
```css
/* Large Title 34 pt, трекинг +0.4 pt */
letter-spacing: calc(0.4 / 34 * 1em);   /* ≈ 0.0118em */
/* Body 17 pt, трекинг −0.43 pt */
letter-spacing: calc(-0.43 / 17 * 1em); /* ≈ -0.0253em */
```

### 3.3 Вес несёт иерархию, а не важность

ЗКН-Д024 уже это сказал, референсы подтверждают: **заголовок строки списка —
17 обычным**, не полужирным. Полужирный в списке из шести строк делает
полужирными все шесть, и иерархии не остаётся.

Полужирный (600–700) — только:
крупный заголовок экрана · заголовок карточки · активный сегмент ·
кнопка-действие · заголовок алерта.

## 4. Геометрия

### 4.1 Врезки

| Что | pt | Источник |
|---|---|---|
| Карточка группы от края экрана | **16** | 📐 `apple_id` с.4 — левая кромка ровно 48 px |
| Поле внутри карточки | **16** | 📐 |
| Поле поиска от края | **20** | 📐 `apple_dev` с.2 — слева 20.0, справа 20.0 |
| Крупный заголовок от края | **20** | 📐 `apple_dev` с.1 |
| Полка/карусель — левый край первой карточки | 16 | 📐 `apple_dev` с.2 |

📐 **Перепись врезок — две, и они по назначению:**

| Врезка | Модальна у | Что за экраны |
|---|---|---|
| **16 pt** | Fitness (14 кадров) · Calendar (6) · Dev (6) · ID (5) · Files | **Сгруппированные списки и карточки** |
| **20 pt** | Music (8) · TV (10) · Wallet (2) · Fitness (5) | **Витрины, медиа, крупные заголовки** |

Правило: **список — 16, витрина — 20.** Смешивать на одном экране нельзя:
разная врезка читается как разная принадлежность там, где её нет.

### 4.2 Скругления

📐 **Проверено на четырёх продуктах одним методом (19.07.2026).** Профиль угла
снимается субпиксельно — граница берётся там, где заливка пересекает 50 %
между холстом и карточкой, — и подгоняется МНК двумя моделями: дугой
окружности и суперэллипсом `|x/R|ⁿ + |y/R|ⁿ = 1`.

| Кадр | Дуга R | RMSE | Суперэллипс R | n | RMSE |
|---|---|---|---|---|---|
| `apple_id` с.1 | 24.17 pt | 1.03 px | 28.67 pt | **2.5** | 0.64 px |
| `apple_files` с.1 | 24.17 pt | 1.08 px | 28.67 pt | **2.5** | 0.64 px |
| `apple_notes` с.9 | 25.83 pt | 1.13 px | 29.67 pt | **2.4** | 0.68 px |
| `apple_wallet` с.3 | 25.83 pt | 1.08 px | 30.67 pt | **2.5** | 0.33 px |

Суперэллипс ложится точнее дуги **во всех четырёх случаях** — в 1.6–3.3 раза
по RMSE, — и показатель держится на **n ≈ 2.4–2.5**. Вывод «не дуга,
непрерывное скругление» подтверждён.

⚠️ **Правка чисел.** Прежняя запись опиралась на один кадр (`apple_id` с.4) и
на довод «круг дал бы 19.3 px на 20-м пикселе, замер даёт 24». Замер 24 был
снят по точному совпадению цвета — такой способ завышает отступ примерно на
2 px, потому что считает антиалиасинг холстом. Субпиксельный замер того же
угла даёт **22.1 px**, а не 24. Довод был верен по направлению и завышен по
величине; радиус **22.3 pt заменён на 24–26 pt** (эквивалент дуги) —
и это, в отличие от прежнего числа, сходится с ЗКН-Д018, где карточка
зафиксирована на 24.

⚠️ n ≈ 2.5 — это **мягкая** непрерывность, а не «яблочный сквиркл» из
презентаций, где показатель называют 4–5. Числа сняты с кадров, а не
с презентаций.

Замер воспроизводится, а не берётся на слово — инструмент лежит в дереве:

```bash
python3 tools/assets/offload.py restore --class ios26-refs   # мастера с зеркала
python3 tools/ios26-corner.py /tmp/apple_files.pdf           # сам найдёт карточку
```

| Элемент | Радиус | Источник |
|---|---|---|
| **Карточка группы** | **24** | 📐 24.2–25.8 pt по четырём продуктам · ЗКН-Д018 фиксирует 24 |
| Меню, лист, алерт | 20 | 📐 `apple_files` с.11 · `apple_dev` с.7 |
| Карточка-герой с изображением | 16 | 📐 `apple_dev` с.2 |
| Кнопка, чип, поле | **капсула** = `height / 2` | 📐 |
| Плавающий таб-бар | **капсула** | 📐 `apple_files` с.4 — профиль совпал с окружностью `h/2` до пикселя |
| Плитка иконки 29 pt | 7 | 🍎 |

```css
border-radius: 24px;
/* Точное непрерывное скругление там, где браузер умеет: */
@supports (corner-shape: squircle) {
  .card { corner-shape: squircle; border-radius: 16px; }
}
```
⚙️ Круглый `border-radius: 24px` визуально занимает ту же зону, что непрерывный
16 pt, — поэтому **в обычном CSS берётся 24**, а не 16. Подмена радиуса без
подмены формы даёт «пузырь» вместо яблочного угла.

**Правило концентричности** 🍎: вложенный элемент получает
`R_внутр = R_внешн − отступ`. Карточка 24, поле 16 → миниатюра внутри 8.
Одинаковый радиус у вложенных углов читается как ошибка вёрстки.

**Два кластера низа — это не разброс, а две разные детали на одной оси.**
Раздел 12 полгода держал вопрос «почему низ бара то 21.3, то 28.3» и гадал про
тип экрана и мини-плеер. Ответ проще: у продуктов с таб-баром деталь высотой

**48.0 pt**. Обе центрированы по одной линии — 52.00 pt над низом экрана.
21.0 + 31 = 52; 28.0 + 24 = 52. Мерить надо было середину, а не кромку.

на деле он в кластере таб-бара (21.0 / 62.0, 11 кадров). И сами числа 21.3 и
28.3 завышены на треть пикселя: кромка бралась по яркости с ореолом стекла,
а не по материалу. Чистые значения — 21.0 и 28.0.

справа через зазор 12 pt (📐 `apple_dev`, `apple_files`). Это язык iOS 26:
поиск — не пятый таб, а действие.

## 6. Материал — Liquid Glass

🍎 Материал iOS 26 — **линза**, а не матовая плёнка. Он преломляет и
**увеличивает** то, что под ним, даёт спектральную кайму по кромке и блик,
едущий за прокруткой. Это уже записано в ЗКН-Д016 и реализовано в таб-баре;
здесь — общее правило и рецепт для остальных поверхностей.

### 6.1 Слои (порядок обязателен)

```
1. backdrop   — blur + saturate  (материал)
2. tint       — белый с малой альфой (тон)
3. refraction — кольцо-маска по кромке: увеличение + аберрация
4. highlight  — inset-блик сверху (спекуляр)
5. shadow     — внешняя тень (отрыв от холста)
```

### 6.2 Рецепт CSS (тёмная тема)

```css
.glass {
  background: rgba(255, 255, 255, 0.06);
  -webkit-backdrop-filter: blur(24px) saturate(180%);
          backdrop-filter: blur(24px) saturate(180%);
  box-shadow:
    inset 0 0.5px 0 rgba(255, 255, 255, 0.18),   /* спекуляр сверху */
    inset 0 -0.5px 0 rgba(255, 255, 255, 0.06),  /* нижняя кромка   */
    0 8px 24px rgba(0, 0, 0, 0.40);              /* отрыв           */
  -webkit-user-select: none;                      /* ЗКН-Д016        */
}
@supports not ((backdrop-filter: blur(1px)) or (-webkit-backdrop-filter: blur(1px))) {
  .glass { background: #181818; }                 /* 📐 замер бара   */
}
```

Соответствия толщин 🍎 → ⚙️:

| Материал Apple | blur | tint (тёмная) |
|---|---|---|
| ultraThin | 12 px | `rgba(255,255,255,0.04)` |
| thin | 18 px | `rgba(255,255,255,0.05)` |
| **regular** | **24 px** | `rgba(255,255,255,0.06)` |
| thick | 34 px | `rgba(255,255,255,0.09)` |
| chrome (бары) | 30 px | `rgba(255,255,255,0.07)` |

### 6.3 Где стекло можно и где нельзя

**Можно:** таб-бар · шапка · меню · алерт · кнопка поверх медиа · чип на фото.

**Нельзя:** карточка контента · строка списка · фон секции.
Это ЗКН-Д014, и он не отменяется: стекло — материал **контролов**, а не
заливка **контейнеров**. Залитая стеклом карточка читается как дырка.
Поверхность контента = `--color-bg-2` + волосяная линия.

**Не прозрачно — значит не стекло.** Молочная непрозрачность запрещена
(ЗКН-Д016): сквозь капсулу обязана читаться структура контента.

## 7. Движение

| Действие | Длительность | Кривая |
|---|---|---|
| Нажатие (`scale`, цвет) | 100 ms | `ease-out` |
| Тумблер, сегмент, чип | 200–250 ms | пружина |
| Появление меню / листа | 300 ms | пружина от точки касания |
| Переход экрана | 400–500 ms | пружина |
| Морф материала | 350 ms | пружина |

```css
--ease-ios:     cubic-bezier(0.32, 0.72, 0, 1);   /* пружина без перелёта */
--ease-out-ios: cubic-bezier(0.25, 0.10, 0.25, 1);
--dur-tap: 100ms;  --dur-ui: 250ms;  --dur-nav: 450ms;
```

⚙️ `cubic-bezier(0.32, 0.72, 0, 1)` — принятое приближение системной пружины
iOS (`response ≈ 0.35`, `damping ≈ 0.9`): быстрый старт, мягкая посадка,
без отскока. Отскок (`bounce > 0`) в iOS 26 остался только у морфа материала.

Обязательно:
```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after { animation-duration: .01ms !important;
    transition-duration: .01ms !important; }
}
```

## 13. Как этот документ растёт

```
новый продукт  →  PDF в refs/  →  строка в INDEX.md
                                        ↓
                              покадровая роспись
                                        ↓
                        замер того, чего здесь нет  →  раздел с 📐
                                        ↓
                     противоречие с уже записанным?
                                        ↓
                 ДА → это не «уточнение», это ПРАВКА:
                      старое число зачёркивается, пишется почему
```

Число, снятое с большего числа кадров, побеждает. Число без адреса кадра
удаляется — оно ничьё.
