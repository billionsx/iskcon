# СВОД ЗАКОНОВ ПРИЛОЖЕНИЯ (Конституция)

> **Единственный индекс всех законов приложения.** Один файл. Прежде чем трогать
> текст, карточку, дизайн, навигацию, данные или скрипт — сверься с этим сводом.
> Если закон установлен здесь — он применяется ко **всему** приложению.
>
> Этот документ — **реестр и процесс**. Полнотекстовые формулировки живут в
> файлах-источниках истины (колонка «Источник»); свод их **индексирует**, а не
> дублирует. Правило одного места: деталь правится в источнике, свод хранит ссылку.

---

## 0. Зачем этот свод

Законы в проекте уже были — но **раздроблены** по десятку документов и, главное,
**не принуждались**: инфраструктура принуждения существовала, но осиротела
(`tokens.ts`/`primitives.tsx` — 0 импортов; санитайзер `cleanCardText` подключён к
2 карточкам из 13; статического линтера законов в CI нет вообще). Поэтому один и
тот же закон приходилось повторять устно снова и снова — а устная форма забывается.

**Корень проблемы не в отсутствии законов, а в отсутствии принуждения.** Свод
чинит это одним правилом: *каждый повторяющийся закон обязан подняться по лестнице
принуждения до уровня, на котором нарушение технически невозможно.*

---

## 1. Лестница принуждения (У0 → У5)

Сила закона = уровень, на котором он живёт. Чем выше — тем невозможнее нарушение.

| Ур. | Форма | Кто гарантирует | Слабость |
|----|-------|-----------------|----------|
| **У0** | Память / устно | никто | забывается — **корень боли** |
| **У1** | Документ (`docs/…`) | добрая воля | не читается |
| **У2** | Общий модуль (токен/примитив/компонент) | принятие в коде | можно не импортировать |
| **У3** | Правка данных в D1 | одноразовая | регрессирует при новом контенте |
| **У4** | Санитайзер в рантайме (`cleanCardText`) | код | покрывает только то, что через него прогнали |
| **У5** | Гейт в CI (grep / SQL, fail сборки) | конвейер | — нарушение не может попасть в прод |

**ЗАКОН ЗАКОНОВ (мета-закон).** Терминальное состояние повторяющегося закона —
**У4 или У5**. Закон, оставленный на У0/У1, считается *незакрытым* (баг процесса).
При сомнении выбираем более высокий уровень.

Иерархия при конфликте: **У5 > У4 > У3 > У2 > У1 > У0.** Гейт/санитайзер главнее
документа; документ главнее памяти.

## 2. Жизненный цикл закона (создание → утверждение → установление)

```
ПРЕДЛОЖЕНИЕ → УТВЕРЖДЕНИЕ → УСТАНОВЛЕНИЕ → ПРОВЕРКА → ЗАКРЫТИЕ
```

1. **ПРЕДЛОЖЕНИЕ.** Формулировка + домен + *почему* (какая повторяющаяся боль) +
   целевой уровень принуждения.
2. **УТВЕРЖДЕНИЕ.** Основатель утверждает (одним словом). Закон получает ID
   (`ЗКН-<домен><NNN>`) и строку в §4.
3. **УСТАНОВЛЕНИЕ.** Реализуется на **максимально возможном** уровне:
   - механическое, контекст-независимое → санитайзер (У4) **и/или** CI-grep (У5);
   - смысловое, контекст-зависимое → правка данных в D1 (У3) **+** по возможности
     CI-grep/SQL, ловящий регрессию (У5);
   - визуальное → перенос в общий токен/примитив (У2) **+** линтер против «сырых»
     значений (У5).
4. **ПРОВЕРКА.** Команда проверки (`grep`/SQL/аудит) возвращает **0 нарушений**.
   Идеал — команда вшита в CI (`check-web.yml` / отдельный `laws-lint.yml`).
5. **ЗАКРЫТИЕ.** Закон закрыт, только если: проверка = 0 **И** уровень ≥ У4/У5.
   Иначе — остаётся в §5 «Долги» до механизации.

Отмена/правка закона — тем же циклом (правится строка в §4 и источник).

## 6. Стандарт скрипта / CI (домен Ц)

Любой скрипт или воркфлоу приложения обязан:

- **Идемпотентность** — повторный запуск не ломает состояние (CREATE IF NOT EXISTS,
  INSERT OR IGNORE с оговоркой: FK-нарушения им НЕ подавляются — один сирота убивает батч).
- **Scrub секретов** — `sed -E 's#(ghp_|github_pat_)[A-Za-z0-9_]+#***#g'` на любом выводе.
- **`sh`-совместимость** — шелл по умолчанию `sh`; bash-специфика (массивы, `for ((…))`)
  — только явным `bash file.sh`. `jq` отсутствует → JSON через `python3 -c`.
- **Недеструктивность по умолчанию** — DROP/DELETE на entity/profile-таблицах — только
  осознанно и с бэкапом; деструктивные воркфлоу помечаются и по возможности отключаются.
- **Один короткий однострочный коммит** — многострочные сообщения ломают сериализацию.

## 7. Как пользоваться сводом (для любого исполнителя)

1. **Перед работой** над текстом/карточкой/дизайном/данными/скриптом — открой §4,
   найди домен, прочти релевантные законы и их источники.
2. **Встретил повторяющуюся правку**, которую основатель говорит не впервые →
   это кандидат в закон. Пройди цикл §2, добавь строку в §4, поставь целевой уровень.
3. **Устанавливай на максимальном уровне** (§1). Документ без механизма — это У1, то
   есть *незакрыто*. Тянись к У4 (санитайзер) или У5 (CI-гейт).
4. **Не полагайся на память.** Память = У0 = корень боли. Свод — источник; память лишь
   указывает на него.
