# Активы вне git: два зеркала + индекс (страховка от бана)

Тяжёлые несекретные бинари (фото, видео, аудио, PDF-мастера) **не живут в
репозитории** — они раздувают клон, жгут минуты CI, а git-историю не почистить.
Они выносятся на **два независимых бесплатных зеркала**, а в git остаётся только
крошечный текстовый индекс. Это и есть страховка: бан или пропажа на одном хосте —
не потеря, а переключение (ЗКН-Пл023, ЗКН-Ф025, ЗКН-Пр008).

## Два зеркала (независимые домены отказа)

- **archive.org** — основное холодное хранилище. Щедро, вечно, бесплатно.
- **GitHub Releases** — зеркало. Для публичного репо бесплатно и почти без потолка;
  ассеты релиза лежат на отдельном CDN и **не считаются в размер git** (≤2 ГБ/файл).

Забанят аккаунт archive.org → раздаёт Releases. Снимут релиз → раздаёт archive.org.
Оба мертвы одновременно — крайне маловероятно (разные компании, разные причины бана).

## Индекс истины — `manifest.jsonl` (в git)

Одна строка JSON на «класс» бинарей: `class`, `sha256` tar-архива, `bytes`, число
файлов, `ia_url`, `gh_url`, `source` (первоисточник для перезаливаемого), `ts`.
Плюс `<ident>.files.tsv` — поимённый список с SHA-256 (целостность при распаковке).
Индекс лежит В git и переживает потерю **любого** хоста и даже D1.

Холодному грузу поштучные URL не нужны — нужен воспроизводимый бэкап. Поэтому класс
пакуется в один `tar.gz` и грузится одним файлом на каждое зеркало (мало запросов,
просто восстановить).

## Что делать, если аккаунт archive.org забанили

1. **Узнаем сразу.** `assets-verify.yml` еженедельно (и вручную) HEAD-проверяет оба
   зеркала каждого класса. Мёртвое зеркало → прогон краснеет и шлёт уведомление.
2. **Раздача не падает.** Всё, что вынесено, лежит и на GitHub Releases — доступ
   сохраняется, пока чиним.
3. **Восстанавливаем одной командой.** `offload.py restore --class NAME` скачивает
   tar с живого зеркала, сверяет sha256, распаковывает. Для перезаливаемого (Tilda,
   YouTube-видео) можно перезабрать из первоисточника (`source` в манифесте) и залить
   на новый хост.

## Команды

```bash
# вынести каталог на оба зеркала + записать индекс + убрать из git
python3 tools/assets/offload.py offload --class tilda --path archive/tilda-export --rm

# только бинари по расширению (напр., мастера источников; .txt остаётся)
python3 tools/assets/offload.py offload --class sources --path docs/sources --ext pdf,doc,docx --rm

# проверить живость всех зеркал
python3 tools/assets/verify.py

# восстановить класс с живого зеркала
python3 tools/assets/offload.py restore --class tilda
```

Заливка требует сети (archive.org, uploads.github.com) — гоняется в CI
(`assets-offload.yml`), не из песочницы. Ключи: `IA_ACCESS_KEY`/`IA_SECRET_KEY` +
встроенный `GITHUB_TOKEN`. Всё бесплатное — платных балансов нет (ЗКН-Пр008).

## Что НЕ выносим

Рантайм-активы приложения (`apps/web/public/` — обложки, иконки, шрифты) нужны при
сборке и раздаются самим приложением — остаются в репо. Когда/если их выносить,
понадобится ещё резолвер в воркере (`/asset/<key>` с фейловером на зеркало); для
холодного груза он не нужен. Шрифты (.ttf/.otf/.woff2) под гейт Ф025 не попадают.

## История репозитория

Удаление файла из рабочего дерева **не** уменьшает `.git` — старые версии остаются в
истории (git не забывает). Реально сжать историю может только её переписывание
(`git filter-repo`/BFG) — отдельная аккуратная операция, меняющая все хэши коммитов;
делается осознанно, не на автомате. Вперёд: просто не коммитить бинарь.
