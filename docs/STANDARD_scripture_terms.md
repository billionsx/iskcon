# Стандарт: оформление санскритской / бенгальской терминологии

**Правило.** Любой непереведённый санскритский / бенгальский / хинди **термин** в
проточном тексте приложения набирается шрифтом **Georgia + курсив**
(`var(--font-scripture)`). Это единый стандарт для всех текстов: комментарии
(purport), биографии личностей, описания книг, подписи.

Конвенция — **по стандарту BBT** (Бхактиведанта Бук Траст) / Шрилы Прабхупады:

| Курсивом (термин) | Прямым шрифтом |
|---|---|
| `бхакти`, `дхарма`, `према`, `мокша`, `гуны` | имена: `Кришна`, `Арджуна`, `Чайтанья`, `Бхарата` |
| составные: `карма-йога`, `бхакти-йога`, `маха-мантра`, `нишкама-карма` | названия писаний: «Бхагавад-гита», «Махабхарата» |
| `санкиртана`, `киртана`, `джапа`, `парампара`, `сампрадая` | вошедшие в язык слова: `йога`, `карма`, `гуру`, `мантра`, `аватара`, `нирвана` |

Ключевой нюанс: одиночные `йога` / `карма` / `мантра` — прямым (это лоанворды),
но как часть **составного термина** (`карма-йога`, `маха-мантра`) — курсивом
целиком.

## Как применять

**Единственный канонический маркер — компонент `<Skt>`** (`src/ui/Skt.tsx`),
который рендерит `<i class="skt">`. Класс `.skt` (`src/ui/globals.css`) задаёт
`font-family: var(--font-scripture); font-style: italic`. Семантически `<i>` —
именно иноязычный термин, а не «акцентный курсив», поэтому это и корректная
разметка, и гарантированно единый вид.

```tsx
// 1. Текст, который пишем мы сами (JSX-литералы: описания книг, деки):
<Skt>парампара</Skt>

// 2. Проза из данных (комментарии из D1, биографии) — авто-разметка:
{renderTerms(text)}
```

`renderTerms(text)` прогоняет плоский текст через курируемый глоссарий и
оборачивает **только** технические термины; имена и лоанворды остаются прямыми.
Уже подключён в рендерерах комментариев (`BookDetailPage` — экран и PDF) и прозы
личностей (`ContentDetailPage`).

## Глоссарий — единый источник правды

`src/ui/scripture.ts`:

- `TERMS` — формы терминов, которые **выделяются** (кириллица + IAST, с частыми
  падежными формами).
- `STOP` — имена собственные и лоанворды, которые **не выделяются никогда**
  (страховка для матчера).

Матчер консервативен: совпадение только по **точным** перечисленным формам, по
**границам слов** (Unicode-aware, без lookbehind — ради Safari < 16.4), **длинное
совпадение — первым** (составные термины не дробятся). Никакого матчинга по
основе слова — священный текст не искажается ложными срабатываниями.

**Расширить охват** = добавить форму слова в `TERMS` (а новое имя/лоанворд — в
`STOP`). Иное (`<i>`/`<em>` без класса `.skt`) для терминов **не использовать** —
оно даст курсив в body-шрифте, мимо стандарта.

## Что НЕ трогает стандарт

- Блочные транслитерации целиком (стихи, пословный перевод, цитаты) — у них свой
  механизм (инлайн `font-style: italic` → `var(--font-scripture)` через правило в
  `globals.css`).
- Заголовки (display-шрифт) и центрированные секционные интерлюдии — оставляем
  прямыми, чтобы не смешивать засечки с крупным дисплейным начертанием.
- В PDF-экспорте `--font-scripture` переопределён на Noto Serif (диакритика в
  Puppeteer) — `.skt` наследует это автоматически, вид остаётся единым.
