# Золотой стандарт профиля личности (Entity Gold-Standard Profile)

Авторитетное, проверяемое описание личности: кто это, где упоминается, кем был в разных
воплощениях, биография, вклад. Профиль — **верхний слой** над двумя источниками:
**реестр сущностей** (структурные факты) и **корпус текстов** (ссылки, цитаты, комментарии).

Принцип: источник всего — Кришна (kṛṣṇas tu bhagavān svayam, СБ 1.3.28). Единый стабильный `id`;
имена и связи — атрибуты. Каждое утверждение в «золотом» профиле имеет ссылку на источник.

---

## 1. Структура профиля

Для каждого раздела указан **источник истины**: `[Р]` реестр (есть сейчас) · `[Б]` библиотека
текстов (нужен корпус) · `[К]` ручное курирование.

1. **Идентичность** `[Р]` — `id`; имена RU / EN / IAST; алиасы и эпитеты; однострочное определение.
2. **Положение в таттве и классификация** `[Р]` — vishnu / shakti / shiva / jiva-tattva; тип
   (аватара: lila/guna/yuga/purusha/shaktyavesha · вечный спутник · мудрец · царь · демон · ачарья…).
3. **Воплощения и тождества** `[Р]` — кем является в разных лилах: Кришна-лила ↔ Гауранга Лила
   (`gauranga-lila-identity`), `avatar-of` / `expansion-of` / `shaktyavesha-of`. Прим.: «Бхишма — один
   из восьми Васу (Дьяус)» — пока `[Б/К]`.
4. **Где упоминается** `[Р]` книги (`appears-in`) → `[Б]` точные ссылки (стих/глава) и ключевые места.
5. **Биография / житие** `[Б/К]` — повествование по первоисточникам (ШБ, ЧЧ, лиламрита) и комментариям;
   ключевые события, даты (где применимо).
6. **Семья и окружение** `[Р]` — родители, супруг(а), дети, братья/сёстры; гуру / ученики; спутники;
   духовные братья.
7. **Вклад и учение** `[Б/К]` — что совершил, чему учил, какие труды создал (`author-of`), значение.
8. **Источники** `[Б/К]` — явные ссылки (книга, глава, стих), цитаты ≤ нормы, провенанс.
9. **Метаданные** `[Р]` — `confidence`, `iast_status`, уровень зрелости, дата ревизии, куратор.

## 2. Уровни зрелости профиля (утв. 11.07.2026 — ЗКН-Р003/Р004)

> **ЗОЛОТО = авторитетная мини-книга.** По личности собрана биография полнее, чем в
> любом отдельном источнике на планете. Критерий — **не объём, а ИСЧЕРПАНИЕ**:
> в доступных источниках больше нечего взять.

| уровень | критерий |
|---|---|
| **stub** | есть `id` + имена + таттва (реестр) |
| **bronze** | + классификация, воплощения, книги (`appears-in`), родство — всё из реестра |
| **silver** | + черновая биография, точные стих-ссылки, ключевые цитаты |
| **gold** | **мини-книга**: досье исчерпано (покрытие 100%), полная хронология жизни (явление → детство → встреча → служение → труды → уход → наследие) + учение + труды + наследие; **каждое** утверждение со ссылкой до стиха; IAST verified; аудит = 0 |

`tools/goldforge/goldforge.py audit`. Ручная метка запрещена.

## 3. Хранение

Профиль = структурные факты (живой джойн к реестру) + курируемые тексты. Новая таблица поверх реестра:

```sql
CREATE TABLE entity_profiles (
  entity_id     TEXT PRIMARY KEY REFERENCES entities(id),
  summary       TEXT,         -- однострочное определение (может тянуться из entities.note)
  biography     TEXT,         -- раздел 5 (курируется, со ссылками)
  contribution  TEXT,         -- раздел 7
  level         TEXT NOT NULL DEFAULT 'bronze',   -- stub|bronze|silver|gold
  reviewed_by   TEXT,
  reviewed_at   TEXT,
  updated_at    TEXT NOT NULL DEFAULT (datetime('now'))
);
-- ссылки на стихи (раздел 4/8) — после появления библиотеки:
CREATE TABLE entity_citations (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  entity_id TEXT NOT NULL REFERENCES entities(id),
  work_id   TEXT NOT NULL,    -- 'sb','bg','cc',...  (= entities.id книги)
  ref       TEXT NOT NULL,    -- 'СБ 1.9.40' и т.п.
  kind      TEXT,             -- mention | quote | purport
  note      TEXT
);
```

Идентичность, таттва, воплощения, книги, родство, авторство **не дублируются** в профиле — они
читаются из реестра джойнами. В профиле курируются только проза (биография, вклад) и ссылки/цитаты.

## 4. Зависимость от библиотеки книг — порядок работ

Полная «золотая» глубина требует текстов, но **не всех и не сразу**:

1. **Сейчас:** зафиксировать этот стандарт; сгенерировать профили уровня **bronze** для всех 657
   сущностей из реестра (идентичность, воплощения, книги, родство, линия).
2. **Параллельно — библиотека, 3 ядра:** Шримад-Бхагаватам, Бхагавад-гита, Чайтанья-чаритамрита на
   стих-уровне: `works → divisions (песнь/глава) → verses (стих, перевод, комментарий)`. Эти три дают
   почти все ссылки на личности.
3. **Обогащение:** привязать `entity_citations` (упоминания/цитаты/комментарии) → уровень **silver**.
4. **Курирование:** биография и вклад со ссылками, вычитка → уровень **gold**.
5. **Расширение библиотеки:** остальные книги Прабхупады по мере надобности.

Лицензирование: тексты Прабхупады — копирайт BBT. Для стандарта достаточно **ссылок на стихи и фактов**
(это не ограничено); хранение/показ полных переводов и комментариев публично — отдельный вопрос лицензии
BBT, его подтверждаем до публичной выкладки. Внутренний индекс и цитаты в пределах нормы — рабочий путь.

## 5. Пример профиля — Бхишма (сгенерирован из реестра, уровень bronze)

**1. Идентичность.** `id: bhishma` · RU **Бхишма** · EN Bhishma · IAST **Bhīṣma** · алиасы:
Devavrata, Gangeya.

**2. Таттва и классификация.** jiva-tattva; категории: один из двенадцати махаджан (`maha-jana`),
действующее лицо Махабхараты.

**3. Воплощения и тождества.** В реестре пока не указано. _Ожидает [Б/К]: Бхишма — воплощение Дьяу,
одного из восьми Васу (Махабхарата; СБ)._

**4. Где упоминается.** `[Р]` Бхагавад-гита · Махабхарата · Шримад-Бхагаватам.
_Ожидает [Б]: точные ссылки — напр., уход Бхишмы и его молитвы Кришне (СБ, Песнь 1, гл. 9); поле
Курукшетры (БГ, гл. 1)._

**5. Биография.** _Ожидает [Б/К]: обет безбрачия ради отца Шантану (отсюда имя «Бхишма»); дар умереть
по своей воле; наставления Юдхиштхире о дхарме; уход на ложе из стрел._

**6. Семья и окружение.** `[Р]` отец — Махараджа Шантану; мать — Ганга-деви. (Дети отсутствуют — обет.)

**7. Вклад и учение.** _Ожидает [Б/К]: образец стойкости в обете и преданности; «Бхишма-стути» —
молитвы Кришне в момент ухода._

**8. Источники.** `[Р]` реестр (confidence verified, iast verified). _Стих-ссылки [Б] — после библиотеки._

**9. Метаданные.** confidence: verified · iast_status: verified · уровень: **bronze** · куратор: —

> Видно ровно границу: разделы 1, 2, 4 (книги), 6, 9 — заполнены из реестра уже сейчас; разделы 3, 5, 7
> и точные ссылки в 4/8 — это работа библиотеки и куратора.
