# Стандарт книги (мировой уровень) — Book Standard

Книга в платформе — не PDF и не статья, а **узел того же графа знаний**, что личности и праздники.
Принцип H4 действует и здесь: **ссылка, а не копия; единый источник правды; всё кликабельно**.
`works.id` книги = `id` её сущности-писания в реестре (`entities`), поэтому связи «личность ↔ книга»
(`appears-in`, `author-of`) и слово стиха ↔ личность работают сразу.

---

## 1. Карточки книги (витринные ВБК/ВСК/ВМК + подробная ПК — `docs/STANDARD_cards.md`)

### Витринная карточка (storefront, уровень Airbnb)
Минимум для списков/полок/поиска: обложка, заглавие (RU/EN/IAST), автор, аннотация (1 строка),
бейджи (язык, аудио, число стихов), быстрые действия: **♥ в избранное**, **Читать**, **Слушать**, **Заказать**.

### Подробная карточка (detail)
- **Шапка:** обложка, полные названия, автор (→ карточка личности), категория/жанр, рейтинг, доступные языки.
- **О книге:** описание, структура (песни/главы/стихи), исторический контекст, кому адресована.
- **Действия/сервисы:** Читать онлайн · Аудиокнига · **Заказать с доставкой** · Скачать (если лицензия) ·
  **♥ Избранное** · **Поделиться / QR** · Пожертвовать на печать · Добавить в план чтения.
- **Связи (граф):** ключевые личности книги (→ профили), связанные книги, праздники, лекции, цитаты.
- **Прогресс:** где я читаю, закладки, заметки, стрик чтения.

## 2. Сервисы карточки

| сервис | суть |
|---|---|
| ♥ Избранное (лайк) | `user_favorites` (target_type=`work`/`verse`/`chapter`) |
| Заказать доставкой | `book_orders` (формат, кол-во, адрес, статус); интеграция со складом/магазином храма |

## 3. Навигация внутри книги

`Книга → раздел (песнь/глава/секция) → стих`. Иерархия — таблица `divisions` (через `parent_id`,
произвольная глубина: ШБ = песнь→глава→стих; БГ = глава→стих; Ишопанишад = мантра).
- Хлебные крошки и быстрый переход «к стиху N.N.N».
- Свайп между стихами, оглавление, поиск по книге, перенос закладки между устройствами.
- Режимы: только перевод · перевод+пословный · полный (все слои) · аудио-синхрон.

---

## 4. Стих — пять слоёв (стандарт книг BBT)

Каждый стих (`verses` + `verse_texts` + `verse_tokens`) содержит:
1. **Оригинал (деванагари / бенгали)** — для древних писаний это исходный текст.
2. **Транслитерация (IAST)**.
3. **Пословный перевод (synonyms)** — упорядоченные токены: слово → значение; каждое слово
   кликабельно (см. §6).
4. **Художественный перевод**.
5. **Комментарий (purport)**.

Слой 1–3 — структура писания (для древних текстов открыта); слои 4–5 — авторские (BBT), берутся по
источнику/лицензии (см. §7) с трекингом провенанса в `editions`.

---

## 5. QR-коды и глубокие ссылки

Каждый объект имеет **стабильный id = постоянную ссылку**, на которую генерируется QR:
- книга: `/v/{work}` → `…/v/sb`
- глава/песнь: `/v/{division}` → `…/v/sb.1.9`
- **стих:** `/v/{verse_id}` → `…/v/sb.1.9.40`

`verse_id` (`sb.1.9.40`) — это и первичный ключ, и якорь QR/диплинка. QR ведёт в приложение (deep link)
или на веб с тем же контентом. Делишься стихом — получатель открывает ровно этот стих.

---

## 6. Клик по слову → конкорданс по всему приложению (киллер-фича)

Любое слово в пословном переводе кликабельно. Токен (`verse_tokens`) несёт нормализованную форму
`lemma` и (если слово — имя) ссылку `entity_id` на реестр. По клику:
1. **Где встречается** — все стихи во всех книгах с этой леммой (`verse_tokens.lemma`).
2. **Значение** — словарная статья (`lemmas`): IAST, деванагари, перевод.
3. **Связанная личность** — если слово именует личность (`lemma.entity_id` / `token.entity_id`) →
   карточка личности из реестра, а через неё — её книги, родство, праздники, воплощения.
4. **Семантика** — позже: Vectorize-поиск похожих мест.

Это и есть единство библиотеки и графа: слово стиха → лемма → личность → весь её узел.

## 7. Данные (Cloudflare D1) — см. `library_schema.sql`

`works` (= книга-сущность реестра) · `editions` (язык+права) · `divisions` (иерархия) ·
`verses` (стих = деплинк) · `verse_texts` (перевод+комментарий по редакции) ·
`verse_tokens` (пословно + ссылка на личность) · `lemmas` (индекс конкорданса) ·
`user_favorites` · `book_orders`. Таблицы — **недеструктивные** (CREATE IF NOT EXISTS),
поверх реестра; `works.id`/`verse_tokens.entity_id` ссылаются на `entities.id`.
