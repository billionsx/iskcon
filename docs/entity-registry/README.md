# Реестр сущностей ИСККОН — стартовый словарь (seed)

Это **канонический реестр единиц** платформы: Личности, Писания, и далее места/понятия.
Единый источник истины: всё остальное (книги, бхаджаны, цитаты, истории, поклонение)
ссылается на единицу по её **стабильному `id`**, а не по тексту имени.
Поменял имя в реестре — сменилось всюду.

## Главный принцип
`id` ≠ отображаемое имя. `id` (slug) — неизменный ключ (`krishna`, `arjuna`, `bg`).
Имена (`name_en`, `name_iast`, `name_ru`, `aliases`) — это атрибуты ключа.
Ссылки во всём проекте указывают на `id`.

## Файлы
- `entities_core.csv` — реестр единиц (registry-уровень).
- `relations_core.csv` — граф связей (отдельно, т.к. связи многие-ко-многим).

При сборке БД `entities_core.csv` раскладывается в таблицы `entities` + `entity_names`
(+ типовые расширения), `relations_core.csv` → `entity_relations`.

## Схема `entities_core.csv`
| Колонка | Значение |
|---|---|
| `id` | стабильный slug, неизменен (`krishna`) |
| `type` | `personality` \| `scripture` (далее `place`, `concept`, `mantra`) |
| `category` | категории через `\|` (`vishnu-tattva\|avatar\|bhagavatam`) — единица может быть в нескольких |
| `name_en` | каноническое имя (EN, общепринятое) |
| `name_iast` | с диакритикой (IAST) |
| `name_ru` | каноническое имя (RU, рендеринг ИСККОН) |
| `aliases` | прочие имена/эпитеты через `\|` |
| `confidence` | `verified` (надёжно) \| `review` (требует выверки) |
| `note` | краткая идентификация |

## Схема `relations_core.csv`
`from_id,relation,to_id` — направленный граф.
Словарь связей: `avatar-of`, `expansion-of`, `father-of`, `mother-of`, `son-of`,
`brother-of`, `sister-of`, `wife-of`, `husband-of`, `grandson-of`, `nephew-of`,
`disciple-of`, `guru-of`, `associate-of`, `author-of`, `speaker-of`, `hearer-of`,
`narrator-of`, `appears-in`.

## Статус наполнения
Это **выверенное ядро**, не исчерпывающий список. Длинный хвост (тысячи второстепенных
мудрецов, царей, демонов, спутников) добавляется конвейером:
1. ядро вручную (этот файл);
2. кандидаты из `page_text` и `archive/tilda-export/` (имена уже в текстах);
3. глоссарии/индексы как справочный источник перечней;
4. курирование, `review → verified`.

Каждая правка имени делается **только здесь** (или в БД после миграции) — единый источник истины.
