# Дорожная карта реализации ИСККОН — по вехам (турбинам)

> **Принцип:** каждая веха доводит ОДНУ турбину/мини-приложение до конца так, что приложение
> **уже рабочее и ценное само по себе**. Ценность накапливается. Не «полгода до запуска»,
> а «после каждой вехи есть что показать и чем пользоваться».
>
> Контекст и эталоны мини-приложений — в `PRODUCT_ARCHITECTURE.md` (разделы A–G).

**Как читать веху:** Цель · Турбина и мини-аппы · Что строим · Ценность уже сейчас · Что разблокирует · Данные/тех.

---

## M0 — Фундамент ✅ (готово)
- **Цель:** инфраструктура и каркас.
- **Сделано:** монорепо; Cloudflare D1 (`iskcon-db` + staging); KV (`iskcon-cache`); API `iskcon-api` (Hono+Zod+TS) на `api.gaurangers.com`; CI/CD с авто-смоуком `/v1/health`; схема ядра; сид (6 храмов + календарь).
- **Ценность:** API живой (`/v1/health`, `/v1/centers`, `/v1/calendar`) — есть на чём строить.

## M1 — Храмы мира + парсинг-турбина ⭐ (первая самостоятельная ценность)
- **Турбина:** Т1 Парсинг/Агрегация. **Мини-аппы:** Храмы (Airbnb + Google Maps), Онбординг.
- **Что строим:** карточка храма (витрина уровня Airbnb) + локатор по гео; парсер, автоматически наполняющий все храмы ИСККОН мира (директории центров, расписания, контакты, божества); ручной квиз для claim/уточнения; веб-оболочка на Pages + дизайн-система.
- **Ценность уже сейчас:** «всемирный поиск храмов ИСККОН» — любой человек находит ближайший храм, расписание, контакты, божества. **Самостоятельный ценный продукт** даже без остального.
- **Разблокирует:** локальные события (для календаря), карточку Божества, пожертвования на храм.
- **Данные/тех:** `centers`, `center_programs`, `center_deities`; парсер как Worker + cron; гео-поиск Haversine.

## M2 — Календарь Вайшнавов
- **Турбина:** Т2 Календарь/Титхи. **Мини-апп:** Календарь (Apple + Muslim Pro).
- **Что строим:** движок расчёта Экадаши/праздников/дней явления по гео и часовому поясу; напоминания; «стих/праздник дня»; связь праздник ↔ храм (локальные события) ↔ личность.
- **Ценность:** храмы **+** «что сегодня и что делать» + напоминания → первый ежедневный крючок возврата. Карточки храмов показывают локальные фестивали.
- **Разблокирует:** ежедневные уведомления; привязку личностей (M6); геймификацию постов (M7).
- **Данные/тех:** `festivals`, `center_events`; вычислительный движок календаря.

## M3 — Библиотека + стих дня
- **Мини-апп:** Библиотека (YouVersion + Kindle + Audible). Питает Т3 (граф знаний).
- **Что строим:** чтение книг Прабхупады (ридер уровня YouVersion) + стих дня + аудио. Уважая права BBT — старт через ссылки/лицензию (см. раздел 2 мастер-документа).
- **Ценность:** храмы + календарь + **книги** = настоящее «всё в одном» ядро. Стих дня усиливает ежедневный возврат.
- **Разблокирует:** связи книги ↔ личности ↔ праздники (граф); образование (M9); семантический поиск.
- **Данные/тех:** `books`, `authors`, `book_editions`; ридер на Pages; (опц.) Vectorize по тексту.

## M4 — Храм в кармане + аккаунт преданного ⭐ (превращается в ежедневный спутник)
- **Турбина:** П1 Садхана. **Мини-аппы:** Храм-в-кармане (Insight Timer + Calm), Профиль (Apple Health), Даршан дня.
- **Что строим:** карточка Божества + даршан + джапа-счётчик + ежедневный цикл практики + лента «Даршан дня»; **аккаунт преданного** (Better Auth/JWT — для персонализации, стриков, избранного). Включаем `JWT_SECRET`.
- **Ценность:** теперь это **ежедневный духовный спутник** поверх храмов+календаря+книг (даршан, джапа, стих). Резкий рост удержания.
- **Разблокирует:** геймификацию пути (M7); персонализированную ленту; локальные пожертвования (привязка к аккаунту).
- **Данные/тех:** `users`, `user_japa`, `user_saved`, `center_deities`; auth-таблицы Better Auth в D1.

> **M1–M4 = золотой модуль одного храма** (храм + календарь + книги + ежедневная садхана). Это эталон, который доводим до мирового уровня и затем масштабируем мульти-тенантно.

## M5 — Медиа + Krishna radio + Live
- **Турбина:** Т3 (медиа-узлы). **Мини-аппы:** Медиа (Spotify + YouTube + Twitch), Krishna radio, «Сейчас в эфире».
- **Что строим:** библиотека лекций/киртанов + непрерывное аудио 24/7 + хаб живых трансляций даршанов/фестивалей.
- **Ценность:** большое время-в-приложении (аудио фоном) + присутствие на фестивалях мира в реальном времени.
- **Данные/тех:** `media`, `livestreams`; R2/Stream для медиа; интеграции YouTube/стримов.

## M6 — Киртан/мантры + Личности (энциклопедия)
- **Турбина:** Т3 Граф знаний (+ Vectorize). **Мини-аппы:** Киртан (Smule + Genius), Личности (Wikipedia + граф), Ассистент (Perplexity).
- **Что строим:** киртаны/бхаджаны со словами+переводом и пением; мантры/пранамы; энциклопедия личностей как граф (день явления → календарь, лилы → книги, связи → личности, где поклоняются → храмы); семантический поиск «Спроси о КК».
- **Ценность:** контент-глубина, которой нет ни у кого; всё связано в граф; можно «спросить что угодно».
- **Данные/тех:** `personalities`, `personality_relationships`, `glossary_terms`, киртан-тексты; Vectorize-индекс по книгам/лекциям/личностям.

## M7 — Путь преданного (геймификация) + сообщество/сангха
- **Турбины:** П2 Путь, П3 Сева/сангха. **Мини-аппы:** Путь (Duolingo + Habitica), Сообщество (Discord/Circle + Meetup).
- **Что строим:** геймифицированная лестница шраддха→према, квесты, стрики, коллективные челленджи (Джапатон, марафоны); конгрегация, наставник/куратор, регистрация на события, доска севы.
- **Ценность:** глубокая вовлечённость + социальное удержание — возвращаются ради сангхи и прогресса.
- **Данные/тех:** `user_path`/прогресс, `quests`, `challenges`, `congregation`, `mentors`, `seva_tasks`, `events_rsvp`.

## M8 — Пожертвования + Food for Life + магазин
- **Мини-аппы:** Пожертвования (Tithe.ly/Pushpay + Patreon + GoFundMe), FFL (GoFundMe + Meetup), Магазин (Amazon/Shopify).
- **Что строим:** пожертвования на реквизиты конкретного храма + глобальные кампании + дашборд импакта; директория FFL + спонсирование прасада; магазин (требует R2 + платежи).
- **Ценность:** экономический контур — храмы получают донаты, кампании (TOVP), импакт прозрачен; коммерция.
- **Данные/тех:** `donation_campaigns`, `donations`, `memberships`, `ffl_chapters`, `products`, `orders`; Stripe Connect/локальные шлюзы; R2 для товаров.

## M9 — Образование, ятра, матримония, дети, языки + нативное приложение
- **Мини-аппы:** Образование (Coursera + Duolingo), Ятра (izi.TRAVEL), Матримония (Dil Mil), Дети (Epic!), Языки (Duolingo).
- **Что строим:** систематические курсы (Бхакти-шастри…); гайды по паломничеству + аудиогид на месте; вайшнавская матримония; детский/гурукула-контент; санскрит/бенгали; **нативное мобильное приложение** (Expo) на общем API.
- **Ценность:** покрытие всех аудиторий (семьи, молодёжь, паломники, ученики) + полноценное мобильное приложение.

## M10 — Официальный запуск и партнёрства
- **Что делаем:** партнёрство с GBC/BBT; масштабирование онбординга на все храмы; пайплайн переводов; кастомные домены для франчайзи; подключение главного домена `iskcone.com`; официальное позиционирование.
- **Ценность:** из «лучшего в мире продукта одного храма» — в официальную глобальную платформу ИСККОН.

---

## Зависимости и принцип релиза
- M1 не зависит ни от чего (кроме M0). M2 опирается на M1 (локальные события). M4 требует аккаунта (Better Auth). M6 опирается на M3 (граф по книгам). M8 требует R2 + платежей. Нативка (M9) — поверх стабильного API.
- После **каждой** вехи — релиз: всегда есть рабочий ценный продукт. Это снижает риск и даёт ранних пользователей уже с M1.
