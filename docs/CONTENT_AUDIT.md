# Аудит контента iskcone.com и план переноса

Статус: исходный сайт `iskcone.com` (платформа Tilda, проект «ISKCON ONE LOVE»,
неофициальный — совпадает с нашим позиционированием) полностью выгружен в нашу базу
через публичный краулер (без API Тильды, мимо лимита 150/час).

## Что выгружено (сырой слой)
- **170 страниц**, **1279 уникальных изображений** (5487 ссылок).
- `tilda_pages` — по каждой странице: `alias`, `title`, `descr`, полный `html`, `images_json`, `img_count`.
- `tilda_assets` — плоский список всех картинок с привязкой к странице.
- Параллельно: воркфлоу `tilda-sync` (по API-ключам, scoped на проект iskcone) — канонический
  источник, обновляется ночью; краулер `iskcone-crawl` — быстрый резерв.

## Сущности → базы данных
Уже есть: `centers`, `center_programs`, `festivals`, `center_events`, `deities`, `center_deities`.
Добавляем (из переноса): `content_items` (единый каталог: личности/статьи/центры/витрины — slug, lang,
type, name, subtitle, hero_image), далее по мере парсинга — `quotes`, `books`, `prayers`,
`personality_relations` (граф гуру/ученик/спутник), `endorsements`, `stats`.

## Шаблоны «продуктов» (переиспользуемые типы страниц)
1. Личность (имя, эпитеты, цитаты со ссылками на писания, связи) — герой-фича.
2. Центр (карта, расписание, пожертвования, контакты).
3. Книга (часть, глава, стихи, стих, обложка, автор).
4. Молитва/бхаджан (3-слойный текст + аудио + автор).
5. Медиа (плеер).
6. Статья («dāsa»).
7. Пилар-хаб (6 «продуктов» бхакти: Мантра/Книги/Божества/Прасад/Садху/Дхама).
8. Праздник/день календаря.
