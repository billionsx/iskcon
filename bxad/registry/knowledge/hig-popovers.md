# знание · `hig-popovers`
Источник: https://developer.apple.com/design/human-interface-guidelines/popovers
Домены мандата: popup
Нормативных положений: 16 (детерминированная выжимка, не пересказ)


## без раздела
- Use a popover to expose a small amount of information or functionality.
- Make sure a popover’s arrow points as directly as possible to the element that revealed it.
- Use a Close button for confirmation and guidance only.
- If multiple selections are possible, make sure the popover remains open until people explicitly dismiss it or they click or tap outside its bounds.
- Always save work when automatically closing a nonmodal popover.
- Never show a cascade or hierarchy of popovers, in which one emerges from another.
- Don’t show another view over a popover.
- Make sure nothing displays on top of a popover, except for an alert.
- Avoid making a popover too big.
- If necessary, the system can adjust the size of a popover to ensure it fits well in the interface.
- If you adjust the size of a popover, animate the change to avoid giving the impression that a new popover replaced the old one.
- Avoid using the word popover in help documentation.
- For example, instead of “Select the Show button at the bottom of the popover,” you might write “Select the Show button.” Avoid using a popover to show a warning.
- If you need to warn people, use an instead.
- Avoid displaying popovers in compact views.
- Reserve popovers for wide views; for compact views, use all available screen space by presenting information in a full-screen modal view like a sheet instead.
