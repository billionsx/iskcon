# Кузница золота (goldforge)

Конвейер **онлайн-книги о личности**. Одна команда на любую из 730 личностей:

```sh
python3 tools/goldforge/goldforge.py forge jiva-goswami
```

    паспорт имён → жатва по 7 каналам → сборка книги → ГЕЙТ → запись в D1

В песочнице сеть закрыта — каналы k3/k5/k6/k7 работают в CI:
воркфлоу **`pkl-forge.yml`** (Actions → pkl-forge → entity_id).

## Семь каналов жатвы (ЗКН-П001)

| | канал | откуда |
|---|---|---|
| **k1** | книги приложения | D1: 28 книг, 68 816 стихов — перевод **и** комментарий + `quotes` |
| **k2** | архив библиотеки | `docs/sources` (69 МБ, RU + EN) |
| **k3** | книги онлайн | vanisource · vedabase · granthamandira |
| **k4** | бхаджаны приложения | D1: 339 молитв, 1 989 куплетов |
| **k5** | бхаджаны онлайн | kksongs и другие песенники |
| **k6** | Википедия | ru + en |
| **k7** | сайты ИСККОН | vaniquotes · vanipedia · iskcon.org · Desire Tree · gaudiya-history · harekrishna.ru |

Внешние источники — в `sources.json`. Добавил строку → канал сам её опрашивает.
Источник, который лёг или сменил разметку, **не роняет жатву**: канал честно
печатается как «НЕ СОБРАН».

## Три уровня уверенности (ЗКН-П009 — ничего не выбрасывается)

| уровень | признак | судьба |
|---|---|---|
| **strong** | точное имя с титулом («Джива Госвами») | в книгу |
| **candidate** | голое имя («Джива») | в бриф куратору |
| **homonym** | нарицательное («джива» = душа) | в досье, не в книгу |

Точное имя проверяется **первым** — и это спасает жатву: стем «джива» стоит в
тысячах комментариев Шримад-Бхагаватам как «душа». Узкий фильтр убил бы материал,
широкий утопил бы карточку в чужом тексте.

## Почему цитате можно верить

**Цитата рождается в БД, а не в голове.** Текст стиха, комментария и куплета
тянется из D1 и кладётся в карточку без единой правки (ЗКН-П002 · ЗКН-П012).
Чтобы нарушить дословность, надо специально сломать код.

**Внешний источник не даёт дословной цитаты.** Википедия, сайты ИСККОН, онлайн-книги
и вторичная литература дают ФАКТ — он идёт в прозу со ссылкой, но никогда в кавычки
как шастра (ЗКН-П004). Дословный блок появляется только когда книга внесена в
приложение и у цитаты есть `to`.

## Гейт (все проверки испытаны на живых нарушениях)

- канон имён · полное имя героя · нет «;» · нет прямого апострофа в прозе
- первый таб — «Вклад в Гауранга Лилу» (крит. 10)
- 0 дублей заголовков · 0 дублей ссылок · 0 битых `to`
- **дословность**: каждая цитата сверена с D1 подстрочно
- **ноль фабрикации**: каждое имя собственное и число авторской прозы обязано
  лежать в досье. Не лежит → сборка падает
- **храповик**: новая книга не беднее старой (секции и цитаты не убывают)

Вердикт (`bronze`/`silver`/`gold`) выносит **гейт**, а не человек (ЗКН-Р004).

## Связующая проза (необязательно)

Есть `ANTHROPIC_API_KEY` — связки `p` пишет модель, получая ТОЛЬКО материал
секции; её слово проверяет гейт содержания. Нет ключа — связки фактические
(«Стихов о Дживе Госвами в этой главе: 4. Повествует Кришнадас Кавираджа Госвами»),
книга всё равно полна и законна.

## Стадии по отдельности

```sh
goldforge.py passport <id>              # имена: entity_names + склонения + титулы
goldforge.py harvest  <id> [--offline]  # 7 каналов → docs/dossiers/<id>.dossier.json
goldforge.py compose  <id> [--keep-old] # книга → data/pkl/<id>.json + бриф куратору
goldforge.py gate     <id>              # audit-to-zero
goldforge.py publish  <id> [--dry]      # запись в D1 (level ставит гейт)
```
