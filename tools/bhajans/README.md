# Загрузчик бхаджанов

Кладёт бхаджаны в D1 (`prayers` + `prayer_verses`) в эталонной форме.
**Источник-агностик**: знает только нормализованный JSON, не знает, откуда он взялся.

```
парсер источника  ─►  payload.json (эта схема)  ─►  bhajan_loader.py  ─►  D1
   (свой на каждый                                    (один на всех)
    источник)
```

## Запуск

```bash
# проверка + план, без записи
python3 tools/bhajans/bhajan_loader.py data/bhajans_payload.json --dry-run

# реальная запись (нужен CF в окружении)
CF=<cloudflare_api_token> python3 tools/bhajans/bhajan_loader.py data/bhajans_payload.json --write
```

Или воркфлоу `bhajans load` (Actions → Run workflow): `mode=dry-run` / `mode=write`.

## Схема payload (массив песен)

| поле | обяз. | смысл |
|---|---|---|
| `slug` | да | канонический, `/ru/bhajans/<author>/<book>/<NN-translit>` |
| `name` | да | кириллическое название |
| `verses` | да | массив `{translit, translation}`, ≥1 |
| `translit_name` | — | latin-транслит названия |
| `author_name` | — | напр. «Бхактивинод Тхакур» |
| `author_slug` | — | напр. `bhaktivinod-thakur` |
| `source_text` | — | напр. «Шаранагати» |
| `category` | — | напр. «Песни ачарьев» |
| `section` | — | напр. «Дайнья — смирение» |
| `ord` | — | порядок внутри книги |
| `hero_image` | — | дефолт — общий ISKCON-арт |

## Что делает загрузчик

- собирает `prayers.text` (служебная строка + транслит-тело), `prayers.translit`,
  `prayers.translation` и построчные `prayer_verses` с подписями
  «Автор · Название · N-й стих» — точно как у эталонного «Гурудева»;
- **матчит на каталог**: заглушка (`is_catalog=1`) заполняется, дубль не плодится
  (UPSERT по slug + полная замена `prayer_verses`);
- **идемпотентен**: повторный прогон даёт тот же результат;
- **валидирует до записи**: пустой/битый куплет → запись всего пакета отменяется;
- **не выдумывает контент** — кладёт ровно то, что в payload.

## Лицензионный гейт для внешних источников

Загрузчик нейтрален. А вот **парсер** чужого сайта — нет.
Прежде чем массово тянуть транслит/перевод со стороннего ресурса, должно быть
основание: контент свободно лицензирован (копилефт / CC / явное «свободно
распространяется») **или** есть прямое разрешение правообладателя.
Public-domain — это бенгальский/санскритский оригинал; конкретный **русский
перевод** — отдельная работа со своим копирайтом. Без основания парсер под
такой источник не пишется.
